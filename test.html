📌 目前測試結果分析與待解決問題

根據測試結果，成功透過 <form> 提交交易，並且在 HiTRUSTpay 商家後台 看到訂單記錄，但仍有以下 兩個主要問題需要解決：

1️⃣ Webhook (merUpdateURL) 是否有被 HiTRUSTpay 呼叫？

🔹 問題

  • HiTRUSTpay 應該會在交易完成後，透過 merUpdateURL (https://isunfa.tw/api/test/payment/update) 回傳交易結果，但目前 後端沒有看到任何請求。
  • 無法確認 HiTRUSTpay 是否真的有發送 Webhook，或者是否被擋住。

🔹 可能原因

  1. Webhook 未觸發
    • HiTRUSTpay 可能未正確發送請求到 merUpdateURL，需要從 HiTRUSTpay 後台確認交易通知狀態。
  2. 請求被擋住
    • 伺服器防火牆或 WAF 可能攔截了來自 HiTRUSTpay 的 Webhook 請求（IP：203.75.242.8）。
  3. 請求格式或 Header 不符合後端驗證邏輯
    • HiTRUSTpay 可能使用了 不同的 Content-Type 或 Header，導致後端 API 拒絕處理請求。

🔹 解決方案

✅ 檢查 HiTRUSTpay 是否有發送 Webhook

  • 從 HiTRUSTpay 後台查看 merUpdateURL 交易通知紀錄，確認 HiTRUSTpay 是否有發送請求。
  • 若後台沒有紀錄，可能 HiTRUSTpay 端有問題，需聯絡客服。

✅ 檢查伺服器日誌 & 設置 Debug Log

  • 在 https://isunfa.tw/api/test/payment/update 後端 API 加入請求日誌，紀錄所有請求 Header & Body： 
app.post('/api/test/payment/update', (req, res) => {
      console.log('Received Webhook:', {
        headers: req.headers,
        body: req.body
      });
      res.send('R01=00'); // 必須回傳這個，否則 HiTRUSTpay 會重發 Webhook
    });
  • 檢查 Nginx / Cloudflare 是否有攔截 Webhook。

✅ 開放 HiTRUSTpay 來源 IP

  • 確保伺服器允許來自 203.75.242.8 的請求：
iptables -A INPUT -s 203.75.242.8 -j ACCEPT

（適用於 Linux 防火牆，若使用 Cloudflare，請在「防火牆規則」內開放）

2️⃣ orderdesc 在 HiTRUSTpay 後台顯示亂碼

🔹 問題

  • orderdesc="測試交易" 這個值 在 HiTRUSTpay 後台顯示為亂碼，可能是 字元編碼問題。

🔹 可能原因

  1. HiTRUSTpay 可能要求 orderdesc 必須是 URL encoded
    • HiTRUSTpay 可能 不支援 UTF-8 字元直接傳送，需要先用 encodeURIComponent 轉換。
  2. HiTRUSTpay 可能只支援 Big5 或其他編碼
    • 需要確認 API 文件，看 HiTRUSTpay 是否只支援 Big5 編碼，而不是 UTF-8。

🔹 解決方案

✅ 嘗試 URL encoding

  • 修改 <form>，使用 JavaScript 先將 orderdesc 進行 URL encoding： 
<input type="hidden" name="orderdesc" value="測試交易" id="orderdesc" />
    <script>
      document.getElementById('orderdesc').value = encodeURIComponent('測試交易');
    </script>

✅ 測試 Big5 編碼

  • 若 HiTRUSTpay 只支援 Big5，可改用 Big5 轉換：
import iconv from 'iconv-lite';
    
    const orderdesc = '測試交易';
    const big5Encoded = iconv.encode(orderdesc, 'big5').toString('hex');
    console.log(big5Encoded);
  • 但 <form> 無法直接處理 Big5，需要後端轉換。

✅ 聯絡 HiTRUSTpay 客服確認

  • 詢問 HiTRUSTpay 是否有指定 orderdesc 需要特定編碼（Big5 / URL encoded）。
